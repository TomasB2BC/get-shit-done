  <action>
Create `get-shit-done/workflows/integrity-check.md` following the GSD workflow pattern from existing workflows (plan-phase.md, verify-work.md).

**Structure the workflow with these steps:**

**Step 0: Project Resolution** -- Copy the standard --project flag handling pattern from plan-phase.md Step 0 verbatim.

**Step 1: Validate Environment and Detect Agent Mode** -- Verify .planning/ exists. Detect AGENT_MODE from config.json. Parse --scope and --fix-all flags from $ARGUMENTS.

**Step 2: Discover Evidence Sources** -- Do NOT hardcode paths. Use discovery:
```bash
# Git history (always available if in a git repo)
git log --oneline -30 2>/dev/null

# Planning files
ls .planning/*.md .planning/phases/*/*.md 2>/dev/null

# Session logs (discover location)
ls .planning/logs/ .planning/sessions/ logs/ 2>/dev/null

# Task logs
ls .planning/tasks/ .planning/todos/ 2>/dev/null
```

**Step 3: Spawn Parallel Recon Probes** -- Spawn 3-4 parallel Task(subagent_type="Explore") probes. Each probe answers ONE narrow question:

1. **State accuracy probe**: Compare STATE.md claims (current phase, plan counts, completion status, last activity) against git log evidence and actual phase directory contents. Write findings to `.scratch/integrity/probe-state.md`.

2. **Roadmap alignment probe**: Compare ROADMAP.md phase plan counts, completion checkmarks, and status claims against actual PLAN.md and SUMMARY.md file counts in phase directories. Write findings to `.scratch/integrity/probe-roadmap.md`.

3. **Cross-reference probe**: Check that @-references in PLAN.md files point to files that actually exist. Check that context file paths in execution_context sections are valid. Write findings to `.scratch/integrity/probe-references.md`.

4. **Decision currency probe** (optional, if PROJECT.md exists): Check PROJECT.md Key Decisions table dates against recent git activity for staleness signals. Write findings to `.scratch/integrity/probe-decisions.md`.

Each probe must follow this output format:
```
GAP: [description of discrepancy]
CLAIM: [what the doc says]
EVIDENCE: [what reality shows]
SEVERITY: [stale | missing | inaccurate | minor]
FIX: [proposed correction]
---
```

**Step 4: Synthesize Gap Report** -- Read all probe output files. Deduplicate gaps. Sort by severity (inaccurate > stale > missing > minor). Present gaps with lettered options:

For each gap:
```
Gap {N}: {description}
  CLAIM: {what doc says}
  EVIDENCE: {what reality shows}

  a) Fix now -- {proposed correction}
  b) Park as todo
  c) Not actually a gap (misread)
  d) Needs investigation
```

**Step 5: Collect Responses** --
- If AGENT_MODE=true OR --fix-all flag: Auto-select "a" (fix now) for all gaps with severity "stale" or "inaccurate". Auto-select "c" for severity "minor". Log via auto-decide.
- If AGENT_MODE=false: Present full gap report. Use AskUserQuestion to collect lettered selections for all gaps at once (e.g., "1a, 2c, 3a, 4b").

**Step 6: Execute Fixes** -- For each gap where user/auto selected "a" (fix now):
- Read the target file
- Apply the proposed correction using Edit tool (surgical, style-matched to existing content)
- Track changed files

For each gap where user/auto selected "b" (park as todo):
- Call /gsd:add-todo or create a todo entry directly

**Step 7: Commit and Report** --
```bash
git add [changed files]
git commit -m "chore: sync planning docs with reality"
```

Display summary:
```
>> Integrity check complete
   Gaps found: {N}
   Fixed: {M}
   Parked as todo: {P}
   Dismissed: {D}
   Needs investigation: {I}
```

**Step 8: Clean Up** -- Remove .scratch/integrity/ probe files.

**Step 9: Return Structured Result** -- Return either:
- `## INTEGRITY CHECK PASSED` -- if no gaps found or all gaps resolved
- `## INTEGRITY CHECK COMPLETE` -- with summary of actions taken
- `## INTEGRITY CHECK BLOCKED` -- if critical gaps need investigation

IMPORTANT DESIGN RULES:
- All human-facing options use LETTERS (a, b, c, d), never numbers
- Probes use Task(subagent_type="Explore") -- NOT general-purpose
- Evidence source discovery is dynamic, not hardcoded paths
- Each probe writes to .scratch/integrity/ (session-ephemeral, cleaned up after)
- Fixes are style-matched: read existing format, match it
- The workflow must work both standalone and when called from elevate-decision Pass 2
