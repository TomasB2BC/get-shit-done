**Title:** "Installing the GSD Hybrid Fork"

**Section 1: Prerequisites**
- Node.js >= 16.7.0
- Git installed and in PATH
- Claude Code installed and working

**Section 2: Pre-Install -- Handle Local Install (CRITICAL)**
This is the most important section. Explain that Claude Code loads .claude/ locally before ~/.claude/ globally. If a project has a local .claude/ with upstream GSD, the global fork install will be shadowed.

Steps:
1. Check for local GSD install: `ls <project>/.claude/commands/gsd/ 2>/dev/null`
2. If present, BACK UP first: `cp -r <project>/.claude/commands/gsd/ <project>/.claude/commands/gsd-backup/` (same for agents/gsd-* and get-shit-done/)
3. Remove local GSD directories (ONLY the GSD-managed dirs, NOT settings.json, CLAUDE.md, hooks, or custom agents):
   - `rm -rf <project>/.claude/commands/gsd/`
   - `rm -rf <project>/.claude/get-shit-done/`
   - Remove ONLY gsd-*.md files from agents: `rm <project>/.claude/agents/gsd-*.md` (do NOT rm -rf the entire agents dir)
4. Explain: after removal, all GSD commands resolve to the global ~/.claude/ install

Include a NOTE: Custom agents that do NOT start with `gsd-` are safe and untouched (e.g., agent-builder.md, cold-email-pipeline-auditor.md). The `gsd-` prefix is reserved for GSD system agents. If you have custom agents with the `gsd-` prefix, rename them before installing.

**Section 3: Install the Fork**
Steps:
1. Clone the repo (or git pull if already cloned): `git clone <repo-url> ~/gsd-hybrid` (use placeholder URL)
2. Ensure you are on the latest commit: `cd ~/gsd-hybrid && git checkout main && git pull`
3. Run the installer: `node bin/install.js --claude --global`
4. Explain what the installer does: copies commands/, agents/gsd-*.md, get-shit-done/, hooks/dist/ to ~/.claude/. Merges hook config into settings.json. Does NOT touch CLAUDE.md, .planning/, or non-gsd agents.

Create `WHATS-NEW.md` at the repo root. This documents every feature this fork adds that upstream GSD does not have. The target audience is a developer already using upstream GSD who wants to know what they gain by switching.

**Title:** "What's New in the GSD Hybrid Fork"

**Section 1: Overview**
Brief paragraph: This fork adds autonomous agent-mode execution, optional Agent Teams orchestration (hybrid mode), cross-project command routing, and decimal phase support to the Get Shit Done framework. All new features are opt-in via configuration. Existing GSD behavior is unchanged by default.

**Section 2: Agent Mode (/gsd:auto)**
- New command: `/gsd:auto [--project <alias>] [--max-phases N] [--single-phase] [--resume]`
- Runs entire milestones autonomously: plan -> execute -> verify cycle per phase
- Auto-decides at all human interaction points (no manual input needed)
- Creates `.planning/AUTO-DISPATCH-LOG.md` for audit trail
- Stop gracefully by creating `.planning/STOP` file
- Requires: `"agent_mode": true` in `.planning/config.json`

**Section 3: Agent Teams Integration (Hybrid Mode)**
- Optional 3-agent debate teams for phase research (optimist / devil's advocate / explorer)
- Optional competing hypotheses for debugging (3 investigators)
- Optional adversarial verification (validator / breaker / reviewer)
- Optional collaborative codebase mapping (4 cross-referencing mappers)
- All off by default -- enable via `"orchestration": "hybrid"` and per-capability toggles in config
- Falls back to classic mode when Agent Teams unavailable
- Config example:
```json
{
  "orchestration": "hybrid",
  "agent_teams": {
    "research": true,
    "debug": true,
    "verification": true,
    "codebase_mapping": true
  }
}
```

**Section 4: Project Aliases (/gsd:register-project)**
- New command: `/gsd:register-project [alias]`
- Register any project directory by name in `~/.claude/projects.json`
- Use `--project <alias>` flag on ALL GSD commands from any directory
- Example: `/gsd:execute-phase 3 --project b2b` runs Phase 3 of the b2b project from anywhere
- New projects auto-registered by `/gsd:new-project`

**Section 5: Decimal Phase Support**
- Phases can be numbered with decimals: `1.1`, `1.2` for sub-phases
- Phase display, sorting, and dispatch all handle decimals correctly
- Existing integer-numbered phases work identically -- no migration needed

**Section 6: Lead-Approval Mode**
- Third autonomy level between classic (all manual) and full-auto
- Classifies decisions by impact (low/medium/high/critical)
- Low-impact decisions auto-resolved; high-impact routed to user via AskUserQuestion
- Recovery via PENDING_APPROVAL.md if session dies during approval wait
- Config: `"autonomy_level": "lead-approval"` in agent_mode_settings

**Section 7: Technical Improvements**
- Auto-dispatch loop rewrite with stuck-loop detection and false-positive guards
- Step 0 project resolution in all 26+ workflow files (enables --project flag everywhere)
- execute-plan.md direct agent-mode detection (more reliable than environment sniffing)
- 5 tech debt fixes for dispatch timing and Step 0 invariants

**Section 8: Before/After Comparison**
Table format:
| Capability | Upstream | This Fork |
|-----------|----------|-----------|
| Autonomous milestone | Not available | /gsd:auto |
| Cross-project commands | Must cd to project | --project <alias> |
| Sub-phases | Integer only | Decimal support (1.1, 1.2) |
| Research depth | Single researcher | Optional 3-agent debate |
| Debug depth | Single debugger | Optional competing hypotheses |
| Verification depth | Single verifier | Optional adversarial pattern |
| Decision autonomy | All manual or all auto | 3 levels (auto-decide, lead-approval, full-auto) |
