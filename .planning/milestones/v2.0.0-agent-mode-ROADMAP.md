# Milestone v2.0.0: agent-mode

**Status:** SHIPPED 2026-02-22
**Phases:** 8-14
**Total Plans:** 17

## Overview

**Goal:** GSD commands run end-to-end without human input when in agent mode

Full design doc: AGENT-MODE-GSD.md
Prerequisite: Phases 1-7 complete (hybrid integration working)

Chain-of-Agents dispatcher spawns fresh Task agents per phase, auto-decide engine replaces AskUserQuestion gates, lead-approval integration adds human oversight for architectural decisions, and developer tooling (decimal phases, project aliases) rounds out the autonomous pipeline.

### Phase 11: Team Research Integration
**Goal:** Wire TeamCreate debate protocol into auto-dispatch when agent_teams.research=true, replacing parallel independent researchers with actual inter-agent debate

**Depends on:** Phase 10 (lead-approval integration complete)

**Plans:** 1 plan

Plans:
- [x] 11-01-PLAN.md -- Wire agent_teams config into gsd-tools.js + auto-dispatch.md (config schema, dispatch header, Task prompt context)

**Details:**
When config has `agent_teams.research: true`, auto-dispatch's plan-phase step should spawn a team research flow (TeamCreate + debate protocol) instead of parallel independent researchers. Currently the agent_teams config toggles exist in settings but auto-dispatch doesn't consume them -- it always falls back to the classic parallel Task pattern. ### Phase 12: Decimal Phase Support
**Goal:** Fix dispatcher increment logic to handle decimal phases (5.1, 5.2) without skipping or breaking the counter

**Depends on:** Phase 10 (independent of Phase 11)

**Plans:** 2 plans

Plans:
- [x] 12-01-PLAN.md -- Add list-phases command to gsd-tools.js (phase directory enumeration with numeric sort)
- [x] 12-02-PLAN.md -- Update auto-dispatch.md dispatch loop for array-based iteration over sorted phase list

**Details:**
The dispatch loop uses integer arithmetic (`PHASE=$((PHASE + 1))`). Decimal phases like 5.1 break the counter -- incrementing from 5 jumps to 6, skipping 5.1. The `find-phase` tool may handle lookup, but the increment and sequencing logic needs to be aware of decimal phases inserted via `/gsd:insert-phase`. Affects auto-dispatch.md and potentially gsd-tools.js phase enumeration.

**Completed:** 2026-02-22

### Phase 13: Project Alias Resolver
**Goal:** Add global `--project` flag to all GSD commands with auto-registration, so users can target sub-projects by alias from anywhere

**Depends on:** Phase 10 (independent of Phases 11-12)

**Plans:** 3 plans

Plans:
- [x] 13-01-PLAN.md -- Core gsd-tools.js functions (resolveProject, registerProject, listProjects) + /gsd:register-project command
- [x] 13-02-PLAN.md -- --project flag in all 28 command files + Step 0 Project Resolution in 26 workflow files
- [x] 13-03-PLAN.md -- Auto-registration in new-project workflow + Step 0 for new-project

**Details:**
Add `resolveProject(alias)` to gsd-tools.js that reads `.planning/projects.json` and returns the planning directory path. All GSD commands accept `--project <alias>` (e.g., `/gsd:execute-phase 3 --project venntel`). Auto-registration: `/gsd:new-project` detects when running inside a repo with an existing root `.planning/` and auto-adds the sub-project to `projects.json`. Alias derived from directory name. Existing projects get a one-time `/gsd:register-project` or auto-discovery scan. This eliminates wasted tokens from agents resolving wrong project context.

**Completed:** 2026-02-22

## Milestone Summary

**Key decisions:**
- Chain-of-Agents dispatcher pattern (fresh Task per phase) solves context degradation
- Auto-decide rule engine in gsd-tools.js (can't generate synthetic text) + inline LLM synthesis in workflows
- Tiered logging verbosity (compact for rule-based, verbose for synthetic/skipped)
- Per-callsite thin if/else branching (not section-level duplication) for agent-mode
- Autonomy level nested in agent_mode_settings (meaningless when agent_mode=false)
- Dispatcher-action-level classification only (architectural vs operational)
- Write-ahead PENDING_APPROVAL.md for session-death recovery (no traditional timeout)
- Array-based phase iteration (PHASES bash array + PHASE_IDX) for decimal phase support
- Alias-keyed object in projects.json for O(1) lookup with relative paths

**Issues resolved:**
- verify-phase dispatch was calling /gsd:verify-work instead of /gsd:verify-phase (TD-1)
- execute-plan.md had no direct agent-mode detection, relied on parent prompt injection (TD-3)
- Stuck loop detection had false-positive risk when CRASH_ACTION was empty (TD-5)
- Step 0 cd timing assumption was undocumented (TD-4)
- auto.md missing --project flag documentation (TD-2)

**Tech debt:** All 5 items from milestone audit closed by Phase 14.
